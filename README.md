# ⛳ Putt Together

**One endless mini-golf course, built hole by hole by the community.**

Built for Reddit's *Games with a Hook* Hackathon on [Devvit Web](https://developers.reddit.com/) with Phaser.

## The hook

You can't just play the course — **you are the course.**

1. **Play** the chain of holes other redditors built. Every hole shows the
   record and a **ghost replay** of the record holder's actual shots racing
   alongside your ball.
2. **Build** the next hole in the editor: walls, sand traps, water hazards.
3. **The gate:** you can only publish a hole after **acing it yourself** —
   one stroke, no exceptions. The server re-simulates your ace through the
   same physics engine, so every hole on the course is provably beatable
   and every record is provably honest.

Playing is fun. Beating a record is better. Watching strangers grind against
*your* hole is the best — creators earn royalty points every time their
hole's record gets broken, and new holes + record breaks are announced as
live comments on the post.

## Mechanics

- **Flick golf** — pull back and release, like a slingshot. Walls bounce,
  sand crawls, water costs a penalty stroke.
- **Ghost racing** — the record run replays as a translucent ball with the
  holder's username. Beat it and the crown is yours (and the comment section
  hears about it).
- **Daily energy** — 10 scored runs a day, refilled every morning, plus a
  🔥 streak for coming back. Out of energy? You can still play — you just
  don't score.
- **Creator economy** — publishing pays more than playing, and records on
  your holes pay you forever.
- **Anti-cheat by physics** — the client never reports a score, only the
  shot vectors. The server re-runs them through the identical deterministic
  simulation. If the math doesn't sink the ball, the score doesn't exist.

## Architecture

```
Reddit post (webview)
├── Phaser 4 client (TypeScript + Vite)
│   ├── WorldScene  — the course map: every published hole + records
│   ├── PlayScene   — deterministic flick physics + ghost replays
│   └── BuildScene  — tile editor with the ace-to-publish gate
└── Hono server (Devvit Web)
    ├── /api/world  — hole chain, records, player state (auto-seeds 9 starters)
    ├── /api/hole   — layouts + ghost data · publish (server-verified ace proof)
    ├── /api/shot   — run validation by re-simulation, records, royalties
    └── Redis       — holes, records, ghosts, per-player energy/streak/points
```

The physics engine lives in `src/shared/` and runs on **both** sides:
the client uses it to animate, the server uses it to verify. One source of
truth, zero trust required.

The nine starter holes were generated by `tools/seed-gen.ts`, which
brute-forces a genuine one-stroke solution for every layout through the real
engine — the seeds obey the same law as the players.

## Development

```bash
npm install
npm run dev      # devvit playtest — live in your test subreddit
npm run deploy   # upload a new version
npm run launch   # submit for review + publish
```

Requires Node 22+ and a Reddit account connected to
[developers.reddit.com](https://developers.reddit.com).
